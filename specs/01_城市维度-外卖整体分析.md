# PAGE-城市维度-外卖整体分析 - 开发设计规范

## 1. 概述
* **页面名称**：城市维度-外卖整体分析
* **数据粒度**：城市级别（按 `日期` + `美团系统城市名称` + `商家类型` 聚合）
* **总列数**：74列（A-BV列）
* **数据来源**：
  * `ads_merchant_pnl_daily`（线上盈亏、订单量、异常单、全量商家数，对应 H-AG、BN-BV 列）
  * `ads_merchant_fee_daily`（服务费率、拼好饭、起送配送费、补贴率统计，对应 AH-AW、BD-BM 列）
  * `mv_subsidy_daily`（代理商补贴分析 B端/C端，对应 AX-BC 列）
  * `dim_organization`、`dim_merchant_type` 维度表

## 2. 报表字段及计算逻辑
* **A-G列 基础维度**：`日期` (report_date)、`城市简称` (city_short_name)、`美团系统城市名称` (city_name)、`城市等级` (city_level)、`区域` (region)、`负责人` (manager)、`商家类型` (merchant_type_group)。
* **H-K列 线上盈亏**：
  * **线上账单收入合计** (H)：`SUM(pnl.revenue_total)`
  * **线上账单成本合计** (I)：`SUM(pnl.cost_total)`
  * **线上账单毛利** (J)：`SUM(pnl.revenue_total + pnl.cost_total)`
  * **线上账单毛利率** (K)：`线上账单毛利 / 线上账单收入合计 * 100` (SQL中使用 `ROUND(..., 2)`)
* **L-Q列 订单量与跨期分析**：
  * **消费订单量** (L)：`SUM(pnl.order_count_consume)`
  * **线上账单订单量** (M)：`SUM(pnl.order_count_bill)`
  * **T-n线上账单订单量** (N)：`SUM(pnl.order_count_tn)`
  * **T+n线上账单订单量** (O)：`SUM(pnl.order_count_tn_plus)`
  * **T-n线上账单订单占比** (P)：`T-n线上账单订单量 / 线上账单订单量 * 100`
  * **T+n线上账单订单占比** (Q)：`T+n线上账单订单量 / 线上账单订单量 * 100`
* **R-AB列 交易商家健康度**：
  * **全量商家数** (R)：`COUNT(DISTINCT pnl.merchant_id)`
  * **商家服务费>0商家数** (S)：`COUNT(CASE WHEN pnl.gtv_original > 0 THEN 1 END)`
  * **线上账单毛利≥0商家数** (T)：`COUNT(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) >= 0 THEN 1 END)`
  * **线上账单毛利<0商家数** (U)：`COUNT(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 THEN 1 END)`
  * **商家服务费>0商家占比** (V)：`S / R * 100`
  * **线上账单毛利≥0交易商家占比** (W)：`T / S * 100`
  * **线上账单毛利<0交易商家占比** (X)：`U / S * 100`
  * **消费GTV=0且商家服务费=0商家数** (Y)：`COUNT(CASE WHEN pnl.gtv_original = 0 AND pnl.revenue_service_fee = 0 THEN 1 END)`
  * **消费GTV=0且商家服务费=0商家占比** (Z)：`Y / R * 100`
  * **消费GTV=0且商家服务费>0商家数** (AA)：`COUNT(CASE WHEN pnl.gtv_original = 0 AND pnl.revenue_service_fee > 0 THEN 1 END)`
  * **消费GTV>0且商家服务费=0商家数** (AB)：`COUNT(CASE WHEN pnl.gtv_original > 0 AND pnl.revenue_service_fee = 0 THEN 1 END)`
* **AC-AG列 亏损毛利明细**：
  * **线上账单毛利<0交易商家毛利合计** (AC)：`SUM(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 THEN (pnl.revenue_total + pnl.cost_total) ELSE 0 END)`
  * **按配送类型拆分** (AD-AG)：以上毛利限制 `ddt.delivery_type_name` 分别为 '代理', '跑腿', '商家配送' 和 '其他'。
* **AH-AO列 服务费率差额分段计数**：
  * 对 `fee_actual_no_phf / gtv_no_phf - commercial_support_rate` 进行分段统计（即 H-AR列 的差额），包含 `(-∞, -10%]`, `(-10%, -5%]`, `(-5%, -1%]`, `(-1%, 0%]`, `(0%, 5%]`, `(5%, 10%]`, `(10%, +∞]`。
  * **商家支持服务费差额<0合计** (AO)：`SUM(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf - fee.fee_std_no_phf) < 0 THEN (fee.fee_actual_no_phf - fee.fee_std_no_phf) ELSE 0 END)`
* **AP-AV列 拼好饭保底收入段**：
  * 对 `phf_fee_actual / phf_completed_orders` 进行分析并计数。
* **AW列 单均配送费差额<0交易商家数**：对 `delivery_income_original / order_count_no_phf - min_delivery_fee < 0` 计数。
* **AX-BC列 合作商补贴分析(B端/C端)**：
  * 分别汇总 B端 和 C端 的 `消费GTV原价` (SUM(gtv_original_price))、`代理商补贴金额` (SUM(agent_subsidy_amount))，并计算 `消费原价合作商补贴率`。
* **BD-BG列 合作商补贴率分段计数**：按 `partner_subsidy_amount / gtv_original` 进行分区间计数。
* **BH-BM列 商代比分段计数**：按 `merchant_subsidy_amount / partner_subsidy_amount` 进行分区间计数。
* **BN-BV列 商家原因异常单**：
  * **商家原因异常单合计** (BN)：拒单、不接单、少餐错餐、投诉、差评等指标求和。

## 3. 核心 Doris 查询 SQL
```sql
WITH pnl_agg AS (
    SELECT
        pnl.report_date,
        org.city_name,
        mtt.merchant_type_group,
        MAX(org.city_short_name) AS city_short_name,
        MAX(org.city_level) AS city_level,
        MAX(org.region) AS region,
        MAX(org.manager) AS manager,
        SUM(pnl.revenue_total) AS 线上账单收入合计,
        SUM(pnl.cost_total) AS 线上账单成本合计,
        SUM(pnl.revenue_total + pnl.cost_total) AS 线上账单毛利,
        SUM(pnl.order_count_consume) AS 消费订单量,
        SUM(pnl.order_count_bill) AS 线上账单订单量,
        SUM(pnl.order_count_tn) AS T减n线上账单订单量,
        SUM(pnl.order_count_tn_plus) AS T加n线上账单订单量,
        COUNT(DISTINCT pnl.merchant_id) AS 全量商家数,
        COUNT(CASE WHEN pnl.gtv_original > 0 THEN 1 END) AS `商家服务费>0商家数`,
        COUNT(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) >= 0 THEN 1 END) AS `线上账单毛利>=0商家数`,
        COUNT(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 THEN 1 END) AS `线上账单毛利<0商家数`,
        COUNT(CASE WHEN pnl.gtv_original = 0 AND pnl.revenue_service_fee = 0 THEN 1 END) AS GTV为0且服务费为0商家数,
        COUNT(CASE WHEN pnl.gtv_original = 0 AND pnl.revenue_service_fee > 0 THEN 1 END) AS `GTV为0且服务费>0商家数`,
        COUNT(CASE WHEN pnl.gtv_original > 0 AND pnl.revenue_service_fee = 0 THEN 1 END) AS `GTV>0且服务费为0商家数`,
        SUM(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 THEN (pnl.revenue_total + pnl.cost_total) ELSE 0 END) AS `毛利<0交易商家毛利合计`,
        SUM(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 AND ddt.delivery_type_name = '代理' THEN (pnl.revenue_total + pnl.cost_total) ELSE 0 END) AS `毛利<0_代理`,
        SUM(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 AND ddt.delivery_type_name = '跑腿' THEN (pnl.revenue_total + pnl.cost_total) ELSE 0 END) AS `毛利<0_跑腿`,
        SUM(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 AND ddt.delivery_type_name = '商家配送' THEN (pnl.revenue_total + pnl.cost_total) ELSE 0 END) AS `毛利<0_商家配送`,
        SUM(CASE WHEN pnl.revenue_service_fee != 0 AND (pnl.revenue_total + pnl.cost_total) < 0 AND (ddt.delivery_type_name IS NULL OR ddt.delivery_type_name NOT IN ('代理', '跑腿', '商家配送')) THEN (pnl.revenue_total + pnl.cost_total) ELSE 0 END) AS `毛利<0_其他配送`,
        SUM(pnl.reject_order_count + pnl.refuse_count + pnl.cancel_no_notify_count + pnl.abnormal_status_count + pnl.delay_count + pnl.wrong_meal_count + pnl.complaint_count + pnl.bad_review_count) AS 商家原因异常单合计,
        SUM(pnl.reject_order_count) AS 商家不接单订单量,
        SUM(pnl.refuse_count) AS 商家拒单订单量,
        SUM(pnl.cancel_no_notify_count) AS 商家取消不告知订单量,
        SUM(pnl.abnormal_status_count) AS 商家异常经营状态订单量,
        SUM(pnl.delay_count) AS 商家配送延迟订单量,
        SUM(pnl.wrong_meal_count) AS 商家少餐错餐订单量,
        SUM(pnl.complaint_count) AS 商家违规投诉订单量,
        SUM(pnl.bad_review_count) AS 差评商家订单量
    FROM ads_merchant_pnl_daily pnl
    LEFT JOIN dim_organization  org ON pnl.organization_id = org.city_id AND pnl.org_version = org.version
    LEFT JOIN dim_merchant_type mtt ON pnl.merchant_type_code = mtt.merchant_type_code
    LEFT JOIN dim_delivery_type ddt ON pnl.delivery_type_l1_code = ddt.delivery_type_code AND ddt.level = 1
    WHERE pnl.report_date BETWEEN :start_date AND :end_date
    GROUP BY pnl.report_date, org.city_name, mtt.merchant_type_group
),
fee_agg AS (
    SELECT
        fee.report_date,
        org.city_name,
        mtt.merchant_type_group,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) <= -0.10 THEN 1 END) AS 服务费率差额_负无穷到负10,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) > -0.10 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) <= -0.05 THEN 1 END) AS 服务费率差额_负10到负5,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) > -0.05 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) <= -0.01 THEN 1 END) AS 服务费率差额_负5到负1,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) > -0.01 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) <= 0 THEN 1 END) AS 服务费率差额_负1到0,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) <= 0.05 THEN 1 END) AS 服务费率差额_0到5,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) > 0.05 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) <= 0.10 THEN 1 END) AS 服务费率差额_5到10,
        COUNT(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf / NULLIF(fee.gtv_no_phf, 0) - fee.commercial_support_rate) > 0.10 THEN 1 END) AS 服务费率差额_10到正无穷,
        SUM(CASE WHEN fee.order_count > 0 AND (fee.fee_actual_no_phf - fee.fee_std_no_phf) < 0 THEN (fee.fee_actual_no_phf - fee.fee_std_no_phf) ELSE 0 END) AS `服务费差额<0合计`,
        COUNT(CASE WHEN fee.phf_completed_orders > 0 AND (fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) - fee.phf_standard_income) < 0 THEN 1 END) AS `PHF保底收入差额<0商家数`,
        COUNT(CASE WHEN fee.order_count > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) <= 3.5 THEN 1 END) AS PHF保底收入_0到3p5,
        COUNT(CASE WHEN fee.order_count > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) > 3.5 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) <= 4.0 THEN 1 END) AS PHF保底收入_3p5到4p0,
        COUNT(CASE WHEN fee.order_count > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) > 4.0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) <= 4.5 THEN 1 END) AS PHF保底收入_4p0到4p5,
        COUNT(CASE WHEN fee.order_count > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) > 4.5 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) <= 5.0 THEN 1 END) AS PHF保底收入_4p5到5p0,
        COUNT(CASE WHEN fee.order_count > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) > 5.0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) <= 6.0 THEN 1 END) AS PHF保底收入_5p0到6p0,
        COUNT(CASE WHEN fee.order_count > 0 AND fee.phf_fee_actual / NULLIF(fee.phf_completed_orders, 0) > 6.0 THEN 1 END) AS PHF保底收入_6p0到正无穷,
        COUNT(CASE WHEN fee.order_count_no_phf > 0 AND (fee.delivery_income_original / NULLIF(fee.order_count_no_phf, 0) - fee.min_delivery_fee) < 0 THEN 1 END) AS `单均配送费差额<0商家数`,
        COUNT(CASE WHEN fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) > 0 AND fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) <= 0.05 THEN 1 END) AS 合作商补贴率_0到5,
        COUNT(CASE WHEN fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) > 0.05 AND fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) <= 0.07 THEN 1 END) AS 合作商补贴率_5到7,
        COUNT(CASE WHEN fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) > 0.07 AND fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) <= 0.10 THEN 1 END) AS 合作商补贴率_7到10,
        COUNT(CASE WHEN fee.partner_subsidy_amount / NULLIF(fee.gtv_original, 0) > 0.10 THEN 1 END) AS 合作商补贴率_10到正无穷,
        COUNT(CASE WHEN fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) > 0 AND fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) <= 1 THEN 1 END) AS 商代比_0到1,
        COUNT(CASE WHEN fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) > 1 AND fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) <= 2 THEN 1 END) AS 商代比_1到2,
        COUNT(CASE WHEN fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) > 2 AND fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) <= 3 THEN 1 END) AS 商代比_2到3,
        COUNT(CASE WHEN fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) > 3 AND fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) <= 4 THEN 1 END) AS 商代比_3到4,
        COUNT(CASE WHEN fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) > 4 AND fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) <= 5 THEN 1 END) AS 商代比_4到5,
        COUNT(CASE WHEN fee.merchant_subsidy_amount / NULLIF(fee.partner_subsidy_amount, 0) > 5 THEN 1 END) AS 商代比_5到正无穷
    FROM ads_merchant_fee_daily fee
    LEFT JOIN dim_organization  org ON fee.organization_id = org.city_id AND fee.org_version = org.version
    LEFT JOIN dim_merchant_type mtt ON fee.merchant_type_code = mtt.merchant_type_code
    WHERE fee.report_date BETWEEN :start_date AND :end_date
    GROUP BY fee.report_date, org.city_name, mtt.merchant_type_group
),
subsidy_agg AS (
    SELECT
        mv.report_date,
        org.city_name,
        mtt.merchant_type_group,
        SUM(CASE WHEN act.activity_business_type = 'B端' THEN mv.gtv_original_price ELSE 0 END) AS 消费GTV_B端,
        SUM(CASE WHEN act.activity_business_type = 'B端' THEN mv.agent_subsidy_amount ELSE 0 END) AS 代理商补贴_B端,
        SUM(CASE WHEN act.activity_business_type = 'C端' THEN mv.gtv_original_price ELSE 0 END) AS 消费GTV_C端,
        SUM(CASE WHEN act.activity_business_type = 'C端' THEN mv.agent_subsidy_amount ELSE 0 END) AS 代理商补贴_C端
    FROM mv_subsidy_daily mv
    LEFT JOIN dim_organization  org ON mv.organization_id = org.city_id AND mv.org_version = org.version
    LEFT JOIN dim_merchant_type mtt ON mv.merchant_type_code = mtt.merchant_type_code
    LEFT JOIN dim_activity_type act ON mv.activity_type_code = act.activity_type_code
    WHERE mv.report_date BETWEEN :start_date AND :end_date
      AND act.activity_type_name IN ('减配送费','折扣商品','商家券','满减活动','使用优惠券')
    GROUP BY mv.report_date, org.city_name, mtt.merchant_type_group
)
SELECT
    p.report_date AS 日期,
    p.city_short_name AS 城市简称,

    p.region AS 区域,
    p.manager AS 负责人,
    p.merchant_type_group AS 商家类型,
    p.线上账单收入合计 AS `线上账单收入合计`,
    p.线上账单成本合计,
    p.线上账单毛利,

    p.线上账单订单量 AS 线上账单订单量,
    p.T减n线上账单订单量 AS T减n线上账单订单量,
    p.T加n线上账单订单量 AS T加n线上账单订单量,
    ROUND(p.T减n线上账单订单量 / NULLIF(p.线上账单订单量, 0) * 100, 2) AS T减n线上账单订单占比,
    ROUND(p.T加n线上账单订单量 / NULLIF(p.线上账单订单量, 0) * 100, 2) AS T加n线上账单订单占比,
    p.全量商家数 AS 全量商家数,
    p.`商家服务费>0商家数` AS `商家服务费>0商家数`,
    p.`线上账单毛利>=0商家数` AS `线上账单毛利>=0商家数`,
    p.`线上账单毛利<0商家数` AS `线上账单毛利<0商家数`,
    ROUND(p.`商家服务费>0商家数` / NULLIF(p.全量商家数, 0) * 100, 2) AS `商家服务费>0商家占比`,
    ROUND(p.`线上账单毛利>=0商家数` / NULLIF(p.`商家服务费>0商家数`, 0) * 100, 2) AS `线上账单毛利>=0交易商家占比`,

    p.GTV为0且服务费为0商家数 AS 消费GTV为0且商家服务费为0商家数,
    ROUND(p.GTV为0且服务费为0商家数 / NULLIF(p.全量商家数, 0) * 100, 2) AS 消费GTV为0且商家服务费为0商家占比,
    p.`GTV为0且服务费>0商家数` AS `消费GTV为0且商家服务费>0商家数`,
    p.`GTV>0且服务费为0商家数` AS `消费GTV>0且服务费为0商家数`,
    p.`毛利<0交易商家毛利合计` AS `线上账单毛利<0交易商家毛利合计`,
    p.`毛利<0_代理` AS `线上账单毛利<0交易商家毛利合计_代理`,
    p.`毛利<0_跑腿` AS `线上账单毛利<0交易商家毛利合计_跑腿`,

    p.商家原因异常单合计 AS 商家原因异常单合计,

    p.商家取消不告知订单量 AS 商家取消不告知订单量,
    p.商家异常经营状态订单量 AS 商家异常经营状态订单量,
    p.商家配送延迟订单量 AS 商家配送延迟订单量,
    p.商家少餐错餐订单量 AS 商家少餐错餐订单量,
    p.商家违规投诉订单量 AS 商家违规投诉订单量,
    p.差评商家订单量 AS 差评商家订单量
FROM pnl_agg p
LEFT JOIN fee_agg f ON p.report_date = f.report_date AND p.city_name = f.city_name AND p.merchant_type_group = f.merchant_type_group
LEFT JOIN subsidy_agg s ON p.report_date = s.report_date AND p.city_name = s.city_name AND p.merchant_type_group = s.merchant_type_group
ORDER BY
    p.city_name,
    CASE p.merchant_type_group
        WHEN '全国KA' THEN 1
        WHEN '区域KA' THEN 2
        WHEN '全国/区域KA' THEN 1
        WHEN '城市商家' THEN 3
        WHEN '闪购' THEN 4
        WHEN '医药商家' THEN 5
        ELSE 99
    END;
```

## 4. 前端 Streamlit-AgGrid 配置要点
* **行号配置**：首列固定为浅色居中 `#` 行号列。
* **数字与格式化**：
  * 金额、订单、商家数等数值列配置整数千分位格式化（如 `线上账单收入合计`、`全量商家数` 等）。
  * 占比、毛利率列使用 `VALUE_FORMATTER_PERCENT_ALREADY` 格式化，在末尾追加 `%`。
* **列过滤与可见性**：
  * 支持「城市简称」、「区域」与「商家类型」多选框在前端过滤数据。
  * 提供指标的列选择器组件，允许动态改变可见列。
* **表头提示**：配置 `headerTooltip`，悬停在 `❓` 指标上时，展示指标的公式和取数血缘。
* **条件格式**：线上账单毛利列应用条件格式化，正值显绿色，负值显红色。
