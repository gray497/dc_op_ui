# PAGE-商家维度-外卖商家线上盈亏 - 开发设计规范

## 1. 概述
* **页面名称**：商家维度-外卖商家线上盈亏
* **数据粒度**：商家级别（按 `日期` + `商家ID` 聚合）
* **总列数**：30列（A-AD列）
* **数据源**：
  * `ads_merchant_pnl_daily`（ADS层，预计算了商户级各项账单收入、成本和订单量）
  * 维度表关联：`dim_organization`、`dim_merchant`、`dim_bd`、`dim_merchant_type`、`dim_delivery_type`、`dim_category`

## 2. 报表字段及计算逻辑
* **A-J列 基础维度**：`日` (report_date)、`外卖组织结构` (organization_name)、`商家名称` (merchant_name)、`商家ID` (merchant_id)、`合作BD` (bd_name)、`商家类型` (merchant_type_name)、`业务品类` (category_name, level=1)、`一级品类` (category_name, level=2)、`一级商家配送类型` (delivery_type_name, level=1)、`区域` (region)。
* **K-M列 线上盈亏主干**：
  * **线上账单收入合计** (K)：各收入子项求和（服务费+一口价服务费+配送费+拼好饭拼单宝）。(ADS中为 `revenue_total`)
  * **线上账单成本合计** (L)：各成本子项求和（活动款+合作商服务费+关爱基金+合作商售后赔付）。(ADS中为 `cost_total`，成本字段在库里存储为负数)
  * **线上账单毛利** (M)：`线上账单收入合计 + 线上账单成本合计` (即 `K + L`)
* **N-P列 单均盈亏指标**：
  * **单均线上账单收入** (N)：`线上账单收入合计 / 消费订单量`
  * **单均线上账单成本合计** (O)：`线上账单成本合计 / 消费订单量`
  * **单均线上账单毛利** (P)：`线上账单毛利 / 消费订单量`
* **Q列 线上账单毛利率**：`线上账单毛利 / 线上账单收入合计 * 100` (即 `M / K * 100`，使用 `ROUND(..., 2)`)
* **R-S列 经营分析**：
  * **消费GTV（原价）** (R)：`gtv_original` (商家在美团系统产生的消费原价交易总额)。
  * **消费订单量** (S)：`order_count_consume` (消费端有效完成订单数)。
* **T-V列 订单与结算周期**：
  * **线上账单订单量** (T)：`order_count_bill` (订单开始日期为报表日的订单数)。
  * **T-n线上账单订单量** (U)：`order_count_tn` (订单开始日期早于完成/账单日的跨前订单量)。
  * **T+n线上账单订单量** (V)：`order_count_tn_plus` (订单开始日期晚于完成/账单日的跨后订单量)。
* **W-Z列 账单收入明细**：
  * **商家服务费** (W)：`revenue_service_fee` (美团向商家收取的服务费)。
  * **一口价服务费** (X)：`revenue_fixed_price_fee` (一口价模式收取的服务费)。
  * **用户配送费** (Y)：`revenue_delivery_fee` (用户付给商家的配送费)。
  * **拼好饭拼单宝** (Z)：`revenue_group_order_fee` (拼好饭活动返款/服务费)。
* **AA-AD列 账单成本明细**：
  * **活动款** (AA)：`cost_activity_fee` (B端/C端营销活动代理商分摊成本)。
  * **合作商服务费** (AB)：`cost_partner_service_fee` (向第三方合作商/加盟商结算费用)。
  * **关爱基金** (AC)：`cost_care_fund` (特殊情况补偿/关爱基金)。
  * **合作商售后赔付费用** (AD)：`cost_after_sale_comp` (客诉或售后纠纷赔付金额)。

## 3. 核心 Doris 查询 SQL
```sql
SELECT
  ads.report_date AS 日,
  org.organization_name AS 外卖组织结构,
  COALESCE(dm.merchant_name, '') AS 商家名称,
  ads.merchant_id AS 商家ID,
  COALESCE(bd.bd_name, '') AS 合作BD,
  COALESCE(mt.merchant_type_name, '') AS 商家类型,
  COALESCE(bc.category_name, '') AS 业务品类,
  COALESCE(pc.category_name, '') AS 一级品类,
  COALESCE(dt.delivery_type_name, '') AS 一级商家配送类型,
  org.region AS 区域,
  ads.revenue_total AS 线上账单收入合计,
  ads.cost_total AS 线上账单成本合计,
  ads.revenue_total + ads.cost_total AS 线上账单毛利,
  ads.revenue_total / NULLIF(ads.order_count_consume, 0) AS 单均线上账单收入,
  ads.cost_total / NULLIF(ads.order_count_consume, 0) AS 单均线上账单成本合计,
  (ads.revenue_total + ads.cost_total) / NULLIF(ads.order_count_consume, 0) AS 单均线上账单毛利,
  (ads.revenue_total + ads.cost_total) / NULLIF(ads.revenue_total, 0) AS 线上账单毛利率,
  ads.gtv_original AS 消费GTV原价,
  ads.order_count_consume AS 消费订单量,
  ads.order_count_bill AS 线上账单订单量,
  ads.order_count_tn AS T减n线上账单订单量,
  ads.order_count_tn_plus AS T加n线上账单订单量,
  ads.revenue_service_fee AS 商家服务费,
  ads.revenue_fixed_price_fee AS 一口价服务费,
  ads.revenue_delivery_fee AS 用户配送费,
  ads.revenue_group_order_fee AS 拼好饭拼单宝,
  ads.cost_activity_fee AS 活动款,
  ads.cost_partner_service_fee AS 合作商服务费,
  ads.cost_care_fund AS 关爱基金,
  ads.cost_after_sale_comp AS 合作商售后赔付费用
FROM ads_merchant_pnl_daily ads
LEFT JOIN dim_organization org ON ads.organization_id = org.organization_id
LEFT JOIN dim_merchant dm ON ads.merchant_id = dm.merchant_id
LEFT JOIN dim_bd bd ON ads.bd_id = bd.bd_id
LEFT JOIN dim_merchant_type mt ON ads.merchant_type_code = mt.merchant_type_code
LEFT JOIN dim_delivery_type dt ON ads.delivery_type_l1_code = dt.delivery_type_code AND dt.level = 1
LEFT JOIN dim_category bc ON ads.business_category_code = bc.category_code AND bc.category_level = 1
LEFT JOIN dim_category pc ON ads.primary_category_code = pc.category_code AND pc.category_level = 2
WHERE ads.report_date BETWEEN :start_date AND :end_date
ORDER BY org.region, org.organization_name, ads.merchant_id;
```

## 4. 前端 Streamlit-AgGrid 配置要点
* **行号配置**：首列固定为浅色居中 `#` 行号列。
* **数据过滤**：支持「外卖组织结构」、「区域」、「合作BD」和「商家类型」等多值过滤。
* **AgGrid 参数**：
  * 金额、订单、商家数采用千分位格式化为整数显示（除单均收入/成本/毛利列保留2位小数外）。
  * 线上账单毛利率列采用百分比格式化，末尾追加 `%`。
* **表头悬停提示**：配置 `headerTooltip`，悬停在带有 `❓` 的表头上显示具体的公式与血缘。
* **条件格式**：线上账单毛利列应用正值显示为绿色，负值显示为红色的条件格式化样式。
