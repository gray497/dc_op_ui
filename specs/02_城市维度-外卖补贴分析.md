# PAGE-城市维度-外卖补贴分析 - 开发设计规范

## 1. 概述
* **页面名称**：城市维度-外卖补贴分析
* **数据粒度**：城市级别（按 `日期` + `美团系统城市名称` + `全国/区域KA` + `B端/C端` + `活动类型` 聚合）
* **总列数**：25列（A-Y列）
* **数据源**：
  * `mv_subsidy_daily`（ADS层补贴预聚合视图，由 `dwd_fact_merchant_subsidy_daily` 每日分区刷新生成）
  * `dim_organization`（匹配组织结构/负责人）
  * `dim_merchant_type`（取全国KA/区域KA等分组名）
  * `dim_activity_type`（取B端/C端类型和活动名称）

## 2. 报表字段及计算逻辑
* **A-I列 基础与补贴维度**：
  * **日期** (A)：报表统计日期。
  * **城市简称** (B)：来自 `dim_organization.city_short_name`。
  * **美团系统城市名称** (C)：来自 `dim_organization.city_name`。
  * **城市等级** (D)：来自 `dim_organization.city_level`。
  * **区域** (E)：来自 `dim_organization.region`。
  * **负责人** (F)：来自 `dim_organization.manager`。
  * **全国/区域KA** (G)：对应 `dim_merchant_type.merchant_type_group`。
  * **B端/C端** (H)：对应 `dim_activity_type.activity_business_type`。
  * **活动类型** (I)：对应 `dim_activity_type.activity_type_name`（包含：'减配送费','折扣商品','商家券','满减活动','使用优惠券'）。
* **J-N列 基础金额汇总**：
  * **消费GTV(原价)** (J)：`SUM(gtv_original_price)`
  * **消费GTV(实付)** (K)：`SUM(gtv_actual)`
  * **代理商补贴金额** (L)：`SUM(agent_subsidy_amount)`
  * **商家补贴金额** (M)：`SUM(merchant_subsidy_amount)`
  * **美团补贴金额** (N)：`SUM(meituan_subsidy_amount)`
* **O-R列 补贴率及商代比**：
  * **消费原价合作商补贴率** (O)：`代理商补贴金额 / 消费GTV(原价)` (即 `L / J`)
  * **消费原价商家补贴率** (P)：`商家补贴金额 / 消费GTV(原价)` (即 `M / J`)
  * **消费原价美团补贴率** (Q)：`美团补贴金额 / 消费GTV(原价)` (即 `N / J`)
  * **商代比** (R)：`消费原价商家补贴率 / 消费原价合作商补贴率` (即 `P / O`，分母为 0 时返回 NULL)
* **S-Y列 活动订单量与金额分析**：
  * **消费订单量** (S)：`SUM(order_count)`
  * **活动订单量** (T)：`SUM(activity_order_count)`
  * **活动订单量占比** (U)：`活动订单量 / 消费订单量`
  * **活动订单原价交易额** (V)：`SUM(activity_order_original_amount)`
  * **活动订单原价交易额占比** (W)：`活动订单原价交易额 / 消费GTV(原价)` (即 `V / J`)
  * **活动订单消费GTV** (X)：`SUM(activity_order_gtv)`
  * **活动订单消费GTV占比** (Y)：`活动订单消费GTV / 消费GTV(实付)` (即 `X / K`)

## 3. 核心 Doris 查询 SQL
```sql
SELECT
    mv.report_date AS `日期`,
    org.city_short_name AS `城市简称`,
    org.city_name AS `美团系统城市名称`,
    org.city_level AS `城市等级`,
    org.region AS `区域`,
    org.manager AS `负责人`,
    COALESCE(mt.merchant_type_group, '未知') AS `全国/区域KA`,
    act.activity_business_type AS `B端/C端`,
    act.activity_type_name AS `活动类型`,
    SUM(mv.gtv_original_price) AS `消费GTV(原价)`,
    SUM(mv.gtv_actual) AS `消费GTV(实付)`,
    SUM(mv.agent_subsidy_amount) AS `代理商补贴金额`,
    SUM(mv.merchant_subsidy_amount) AS `商家补贴金额`,
    SUM(mv.meituan_subsidy_amount) AS `美团补贴金额`,
    CASE WHEN SUM(mv.gtv_original_price) <> 0 THEN SUM(mv.agent_subsidy_amount) / SUM(mv.gtv_original_price) ELSE NULL END AS `消费原价合作商补贴率`,
    CASE WHEN SUM(mv.gtv_original_price) <> 0 THEN SUM(mv.merchant_subsidy_amount) / SUM(mv.gtv_original_price) ELSE NULL END AS `消费原价商家补贴率`,
    CASE WHEN SUM(mv.gtv_original_price) <> 0 THEN SUM(mv.meituan_subsidy_amount) / SUM(mv.gtv_original_price) ELSE NULL END AS `消费原价美团补贴率`,
    CASE WHEN SUM(mv.agent_subsidy_amount) <> 0 THEN SUM(mv.merchant_subsidy_amount) / SUM(mv.agent_subsidy_amount) ELSE NULL END AS `商代比`,
    SUM(mv.order_count) AS `消费订单量`,
    SUM(mv.activity_order_count) AS `活动订单量`,
    CASE WHEN SUM(mv.order_count) <> 0 THEN SUM(mv.activity_order_count) / SUM(mv.order_count) ELSE NULL END AS `活动订单量占比`,
    SUM(mv.activity_order_original_amount) AS `活动订单原价交易额`,
    CASE WHEN SUM(mv.gtv_original_price) <> 0 THEN SUM(mv.activity_order_original_amount) / SUM(mv.gtv_original_price) ELSE NULL END AS `活动订单原价交易额占比`,
    SUM(mv.activity_order_gtv) AS `活动订单消费GTV`,
    CASE WHEN SUM(mv.gtv_actual) <> 0 THEN SUM(mv.activity_order_gtv) / SUM(mv.gtv_actual) ELSE NULL END AS `活动订单消费GTV占比`
FROM mv_subsidy_daily mv
INNER JOIN dim_organization org ON mv.organization_id = org.organization_id
INNER JOIN dim_activity_type act ON mv.activity_type_code = act.activity_type_code
LEFT JOIN dim_merchant_type mt ON mv.merchant_type_code = mt.merchant_type_code
WHERE mv.report_date BETWEEN :start_date AND :end_date
GROUP BY
    mv.report_date, org.city_short_name, org.city_name, org.city_level, org.region, org.manager,
    mt.merchant_type_group, act.activity_business_type, act.activity_type_name
ORDER BY
    org.region, org.city_short_name, mt.merchant_type_group,
    act.activity_business_type, act.activity_type_name;
```

## 4. 前端 Streamlit-AgGrid 配置要点
* **行号配置**：首列固定为浅色居中 `#` 行号列。
* **数字与格式化**：
  * 金额、订单、商家数等数值列配置整数千分位格式化（如 `消费GTV(原价)` 等）。
  * 占比、补贴率、商代比使用占比格式化追加 `%` (除商代比是纯数值外)。
* **多维过滤器**：支持「城市简称」、「区域」、「全国/区域KA」、「B端/C端」多值筛选过滤。
* **表头悬停指标提示**：配置 `headerTooltip`，对接 `config/subsidy_city_metric_help.py`。
